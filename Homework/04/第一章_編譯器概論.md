## 1.1 Introduction

Compilers are among the most remarkable achievements in computer science. They are the bridges that translate human-readable programming languages into the precise machine instructions that computers execute. Every time you write a program in C, C++, Java, or any compiled language, a compiler transforms your high-level intentions into the zeros and ones that drive hardware.

### Why Study Compilers?

Understanding compiler construction provides profound benefits:

| Benefit | Description |
|---------|-------------|
| **Deeper Programming Insight** | You understand what your code truly becomes |
| **Better Debugging Skills** | Error messages and assembly become intuitive |
| **Performance Optimization** | Write code that compilers can optimize effectively |
| **Career Opportunities** | Compiler engineering is a specialized, high-demand field |
| **Language Design** | Understand how to design your own languages |

The evolution of compilers spans over seven decades:

## 1.2 What Is a Compiler?

A **compiler** is a sophisticated software system that translates source code written in a high-level programming language into an equivalent target program, typically in machine code or assembly language.

### 1.2.1 The Translation Process

```
                    ┌────────────────────────────────────────────┐
                    │                                            │
                    │         COMPILER                          │
                    │                                            │
┌──────────┐        │  ┌────────────────────────────────────┐  │        ┌──────────┐
│          │        │  │                                    │  │        │          │
│  Source  │───────►│  │  Analysis    │    Synthesis       │  │───────►│  Target  │

```

### 1.2.2 Compilation vs. Interpretation

Modern systems often blend compilation and interpretation:

| Aspect | Pure Compiler | Pure Interpreter | Hybrid (JIT) |
|--------|--------------|------------------|--------------|
| **Output** | Standalone executable | No output, direct execution | Bytecode + JIT |
| **Execution Speed** | Fastest | Slowest | Fast (after warm-up) |
| **Development Speed** | Slower (compile step) | Faster | Moderate |
| **Error Detection** | Compile-time | Runtime | Mixed |
| **Examples** | C, C++, Rust, Go | Python (classic), Ruby | Java, C#, JavaScript V8 |

## 1.4 Frontend Phases

The **frontend** analyzes the source program, ensuring it conforms to the language specification and building an intermediate representation.

### 1.4.1 Lexical Analysis (Scanning)

Lexical analysis is the first phase of compilation. The **scanner** (or lexer) reads the source code as a stream of characters and groups them into meaningful **tokens**.

#### What Are Tokens?

Tokens are the atomic units of a programming language:

| Category | Examples | Description |
|----------|---------|-------------|
| **Keywords** | `if`, `else`, `while`, `return`, `int`, `float` | Reserved words |
| **Identifiers** | `variableName`, `myFunction` | User-defined names |
| **Operators** | `+`, `-`, `*`, `/`, `==`, `<=`, `&&` | Operations |
| **Literals** | `42`, `3.14`, `"hello"` | Constant values |
| **Punctuation** | `(`, `)`, `{`, `}`, `;`, `,` | Structural markers |
| **Special** | EOF, ERROR | End of file, error tokens |

#### Tokenization Example

```
// Source Code
if (x > 0) {
    return x * 2;
}
```


#### Key Scanner Responsibilities

1. **Ignore whitespace** — Spaces, tabs, newlines (typically)
2. **Remove comments** — `//` and `/* */` style
3. **Track positions** — Line and column numbers for error reporting
4. **Handle errors** — Report invalid characters

### 1.4.2 Syntax Analysis (Parsing)

The **parser** takes the stream of tokens and analyzes them according to the grammar rules of the language. The result is an **Abstract Syntax Tree (AST)**.

### 1.4.3 Semantic Analysis

Semantic analysis ensures that the program makes sense beyond just syntactic structure.

#### Key Activities

| Activity | Description | Example |
|----------|-------------|---------|
| **Type Checking** | Ensure type compatibility | `int x = "hello";` ❌ |
| **Scope Analysis** | Resolve variable references | `x` must be declared before use |
| **Symbol Management** | Track all identifiers | Build symbol table |
| **Constant Folding** | Evaluate constant expressions | `2 + 3` → `5` at compile time |
| **Reachability Analysis** | Check return statements | `int f() {}` missing return |

## 1.5 Backend Phases

The **backend** transforms the intermediate representation into target machine code.

### 1.5.1 Intermediate Representation (IR)

The IR is a platform-independent representation that serves as the bridge between frontend and backend.

#### Properties of Good IR

1. **Abstraction** — Independent of source and target
2. **Expressiveness** — Can represent all language constructs
3. **Analysis-friendly** — Easy to analyze and transform
4. **Low-level** — Close to machine operations

#### LLVM IR Characteristics

```llvm
; LLVM IR Example: Factorial Function

define i32 @factorial(i32 %n) {
entry:
  ; Compare n <= 1
  %cmp = icmp sle i32 %n, 1
  
  ; Branch based on comparison
  br i1 %cmp, label %base_case, label %recursive_case

base_case:
  ; Return 1
  ret i32 1

recursive_case:
  ; Calculate n - 1
  %n_minus_1 = sub i32 %n, 1
  
  ; Recursive call
  %recursive_result = call i32 @factorial(i32 %n_minus_1)
  
  ; Multiply n * factorial(n-1)
  %result = mul i32 %n, %recursive_result
  
  ; Return result
  ret i32 %result
}
```

### 1.5.2 Optimization

Optimization passes transform the IR to improve execution efficiency.

#### Common Optimizations

| Optimization | Description | Example |
|--------------|-------------|---------|
| **Constant Folding** | Evaluate constant expressions | `2 + 3` → `5` |
| **Dead Code Elimination** | Remove unreachable code | `if (false) { ... }` → removed |
| **Common Subexpression** | Eliminate redundant computations | `a = x + y; b = x + y;` → reuse |
| **Loop Unrolling** | Expand loops | `for(i=0;i<4;i++)` → unrolled |
| **Function Inlining** | Replace function calls with body | Small function calls → expanded |

#### Optimization Levels

```bash
# No optimization (debug builds)
clang -O0 -S hello.c -o hello_O0.s

# Basic optimization
clang -O1 -S hello.c -o hello_O1.s

# Standard optimization (default for release)
clang -O2 -S hello.c -o hello_O2.s

# Aggressive optimization
clang -O3 -S hello.c -o hello_O3.s

# Size optimization
clang -Os -S hello.c -o hello_Os.s
```

## 1.7 A Simple Compiler Example

Let's build intuition with a minimal compiler for arithmetic expressions.

### 1.7.1 Simple Expression Language

We define a tiny language for demonstration:

```ebnf
expression := number
           |  expression '+' expression
           |  expression '-' expression
           |  expression '*' expression
           |  expression '/' expression
           |  '(' expression ')'
```

**Example programs:**
- `42` → evaluates to 42
- `2 + 3` → evaluates to 5
- `2 + 3 * 4` → evaluates to 14 (multiplication has higher precedence)
- `(2 + 3) * 4` → evaluates to 20 (parentheses override precedence)

### 1.7.2 Complete Implementation

```cpp
// simple_compiler.cpp
// A minimal expression compiler demonstrating compilation phases

#include <iostream>
#include <string>
#include <vector>
#include <cctype>
#include <stdexcept>
#include <memory>

// ============================================================================
// PHASE 1: LEXICAL ANALYSIS
// ============================================================================

enum class TokenType {
    NUMBER,        // Integer literals
    PLUS,          // +
    MINUS,         // -
    STAR,          // *
    SLASH,         // /
    LPAREN,        // (
    RPAREN,        // )
    END_OF_INPUT,  // Signals end of input
    ERROR          // Invalid token
};

struct Token {
    TokenType type;
    double value;    // For NUMBER tokens
    std::string text; // Original text (for debugging)
    
    Token(TokenType t, const std::string& txt = "", double val = 0.0)
        : type(t), value(val), text(txt) {}
};

class Lexer {
public:
    explicit Lexer(const std::string& input) 
        : input(input), position(0) {}
    
    std::vector<Token> tokenize() {
        std::vector<Token> tokens;
        
        while (position < input.size()) {
            Token token = nextToken();
            tokens.push_back(token);
            
            if (token.type == TokenType::ERROR) {
                throw std::runtime_error("Invalid character at position " 
                    + std::to_string(position));
            }
        }
        
        tokens.push_back(Token(TokenType::END_OF_INPUT));
        return tokens;
    }

private:
    std::string input;
    size_t position;
    
    char currentChar() {
        if (position >= input.size()) return '\0';
        return input[position];
    }
    
    char peekChar() {
        if (position + 1 >= input.size()) return '\0';
        return input[position + 1];
    }
    
    void advance() {
        position++;
    }
    
    void skipWhitespace() {
        while (std::isspace(currentChar())) {
            advance();
        }
    }
    
    Token nextToken() {
        skipWhitespace();
        
        if (position >= input.size()) {
            return Token(TokenType::END_OF_INPUT);
        }
        
        char c = currentChar();
        std::string text(1, c);
        
        if (std::isdigit(c)) {
            return Token(TokenType::NUMBER, text, parseNumber());
        }
        
        advance();
        
        switch (c) {
            case '+': return Token(TokenType::PLUS, text);
            case '-': return Token(TokenType::MINUS, text);
            case '*': return Token(TokenType::STAR, text);
            case '/': return Token(TokenType::SLASH, text);
            case '(': return Token(TokenType::LPAREN, text);
            case ')': return Token(TokenType::RPAREN, text);
            default: return Token(TokenType::ERROR, text);
        }
    }
    
    double parseNumber() {
        std::string numStr;
        
        while (std::isdigit(currentChar())) {
            numStr += currentChar();
            advance();
        }
        
        // Handle decimal part
        if (currentChar() == '.') {
            numStr += currentChar();
            advance();
            
            while (std::isdigit(currentChar())) {
                numStr += currentChar();
                advance();
            }
        }
        
        return std::stod(numStr);
    }
};

// ============================================================================
// PHASE 2: SYNTAX ANALYSIS (Recursive Descent Parser)
// ============================================================================

// Forward declaration
class Parser;

// AST Node base class
class ASTNode {
public:
    virtual ~ASTNode() = default;
    virtual double evaluate() const = 0;
    virtual void print(int indent = 0) const = 0;
};

// Number literal node
class NumberNode : public ASTNode {
public:
    explicit NumberNode(double value) : value(value) {}
    
    double evaluate() const override { return value; }
    void print(int indent = 0) const override {
        std::cout << std::string(indent, ' ') << "Number: " << value << "\n";
    }
    
private:
    double value;
};

// Binary operation node
class BinaryOpNode : public ASTNode {
public:
    BinaryOpNode(char op, std::unique_ptr<ASTNode> left, std::unique_ptr<ASTNode> right)
        : op(op), left(std::move(left)), right(std::move(right)) {}
    
    double evaluate() const override {
        double l = left->evaluate();
        double r = right->evaluate();
        
        switch (op) {
            case '+': return l + r;
            case '-': return l - r;
            case '*': return l * r;
            case '/': 
                if (r == 0) throw std::runtime_error("Division by zero");
                return l / r;
            default: throw std::runtime_error("Unknown operator");
        }
    }
    
    void print(int indent = 0) const override {
        std::cout << std::string(indent, ' ') << "BinaryOp: " << op << "\n";
        left->print(indent + 2);
        right->print(indent + 2);
    }
    
    char getOp() const { return op; }
    const ASTNode* getLeft() const { return left.get(); }
    const ASTNode* getRight() const { return right.get(); }

private:
    char op;
    std::unique_ptr<ASTNode> left;
    std::unique_ptr<ASTNode> right;
};

class Parser {
public:
    explicit Parser(const std::vector<Token>& tokens) 
        : tokens(tokens), position(0) {}
    
    std::unique_ptr<ASTNode> parse() {
        return parseExpression();
    }

private:
    std::vector<Token> tokens;
    size_t position;
    
    Token& currentToken() { return tokens[position]; }
    
    void advance() {
        if (position < tokens.size() - 1) {
            position++;
        }
    }
    
    // Grammar:
    // expression := term (('+' | '-') term)*
    // term := factor (('*' | '/') factor)*
    // factor := number | '(' expression ')'
    
    std::unique_ptr<ASTNode> parseExpression() {
        auto left = parseTerm();
        
        while (currentToken().type == TokenType::PLUS ||
               currentToken().type == TokenType::MINUS) {
            
            char op = (currentToken().type == TokenType::PLUS) ? '+' : '-';
            advance();
            
            auto right = parseTerm();
            left = std::make_unique<BinaryOpNode>(op, std::move(left), std::move(right));
        }
        
        return left;
    }
    
    std::unique_ptr<ASTNode> parseTerm() {
        auto left = parseFactor();
        
        while (currentToken().type == TokenType::STAR ||
               currentToken().type == TokenType::SLASH) {
            
            char op = (currentToken().type == TokenType::STAR) ? '*' : '/';
            advance();
            
            auto right = parseFactor();
            left = std::make_unique<BinaryOpNode>(op, std::move(left), std::move(right));
        }
        
        return left;
    }
    
    std::unique_ptr<ASTNode> parseFactor() {
        Token token = currentToken();
        
        if (token.type == TokenType::NUMBER) {
            advance();
            return std::make_unique<NumberNode>(token.value);
        }
        
        if (token.type == TokenType::LPAREN) {
            advance();
            auto expr = parseExpression();
            
            if (currentToken().type != TokenType::RPAREN) {
                throw std::runtime_error("Expected closing parenthesis");
            }
            advance();
            
            return expr;
        }
        
        throw std::runtime_error("Unexpected token");
    }
};

// ============================================================================
// PHASE 3: INTERPRETATION (Direct AST evaluation)
// ============================================================================

class Interpreter {
public:
    explicit Interpreter(std::unique_ptr<ASTNode> ast) 
        : ast(std::move(ast)) {}
    
    double interpret() {
        return ast->evaluate();
    }
    
    void printAST() {
        std::cout << "AST:\n";
        ast->print(2);
    }

private:
    std::unique_ptr<ASTNode> ast;
};

// ============================================================================
// MAIN: INTEGRATION
// ============================================================================

int main() {
    std::cout << "========================================\n";
    std::cout << "    Simple Expression Compiler v1.0     \n";
    std::cout << "========================================\n\n";
    
    // Test expressions
    std::vector<std::string> testExpressions = {
        "42",
        "2 + 3",
        "10 - 4",
        "2 + 3 * 4",
        "(2 + 3) * 4",
        "10 / 2 + 3",
        "((2 + 3) * (4 - 1)) / 5"
    };
    
    for (const auto& expr : testExpressions) {
        std::cout << "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n";
        std::cout << "Expression: " << expr << "\n";
        std::cout << "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n";
        
        try {
            // Phase 1: Lexical Analysis
            std::cout << "\n[Phase 1] Lexical Analysis:\n";
            Lexer lexer(expr);
            auto tokens = lexer.tokenize();
            
            std::cout << "Tokens: ";
            for (const auto& t : tokens) {
                switch (t.type) {
                    case TokenType::NUMBER: std::cout << "NUM(" << t.value << ") "; break;
                    case TokenType::PLUS: std::cout << "PLUS "; break;
                    case TokenType::MINUS: std::cout << "MINUS "; break;
                    case TokenType::STAR: std::cout << "STAR "; break;
                    case TokenType::SLASH: std::cout << "SLASH "; break;
                    case TokenType::LPAREN: std::cout << "LPAREN "; break;
                    case TokenType::RPAREN: std::cout << "RPAREN "; break;
                    case TokenType::END_OF_INPUT: std::cout << "EOF"; break;
                    default: std::cout << "ERR "; break;
                }
            }
            std::cout << "\n";
            
            // Phase 2: Parsing
            std::cout << "\n[Phase 2] Parsing:\n";
            Parser parser(tokens);
            auto ast = parser.parse();
            
            // Phase 3: Interpretation
            std::cout << "\n[Phase 3] Interpretation:\n";
            Interpreter interpreter(std::move(ast));
            double result = interpreter.interpret();
            
            std::cout << "Result: " << result << "\n";
            std::cout << "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n\n";
            
        } catch (const std::exception& e) {
            std::cerr << "Error: " << e.what() << "\n\n";
        }
    }
    
    return 0;
}
```

## 1.8 Formal Language Basics

### 1.8.1 Regular Expressions for Tokens

Regular expressions provide a formal way to specify token patterns:

| Token | Regular Expression | Examples |
|-------|-------------------|----------|
| Integer | `[0-9]+` | `42`, `123`, `0` |
| Float | `[0-9]+\.[0-9]+` | `3.14`, `0.5` |
| Identifier | `[a-zA-Z_][a-zA-Z0-9_]*` | `x`, `myVar`, `_count` |
| Keyword | `if\|else\|while\|return` | `if`, `else` |
| Operator | `\+\|-\|\*\|/` | `+`, `-`, `*`, `/` |
| Whitespace | `[ \t\n]+` | (ignored) |

### 1.8.2 Context-Free Grammars

Context-free grammars (CFGs) describe the syntax of programming languages:

#### EBNF Notation

```ebnf
program     ::= declaration*
declaration ::= variable-decl | function-decl
variable-decl ::= type identifier ';'
function-decl ::= type identifier '(' params ')' compound-stmt
type        ::= 'int' | 'float' | 'void'
params      ::= param-list | ε
param-list  ::= param (',' param)*
param       ::= type identifier
compound-stmt ::= '{' local-declarations* statement* '}'
statement   ::= expression-stmt | compound-stmt | selection-stmt | iteration-stmt | return-stmt
selection-stmt ::= 'if' '(' expression ')' statement ('else' statement)?
iteration-stmt ::= 'while' '(' expression ')' statement
return-stmt ::= 'return' expression? ';'
expression  ::= var '=' expression | simple-expression
simple-expression ::= additive-expression (relop additive-expression)*
additive-expression ::= term (addop term)*
term        ::= factor (mulop factor)*
factor      ::= '(' expression ')' | var | call | number
call        ::= identifier '(' args ')'
args        ::= expression (',' expression)* | ε
relop       ::= '<=' | '<' | '>' | '>=' | '==' | '!='
addop       ::= '+' | '-'
mulop       ::= '*' | '/' | '%'
```

#### Grammar Visualization


### 1.8.3 Ambiguity and Precedence

Grammars can be ambiguous. The dangling else problem is a classic example:

```c
// Which 'if' does the 'else' belong to?
if (a)
    if (b)
        x = 1;
else
    x = 2;  // Ambiguous!
```

**Resolution:** The `else` binds to the nearest `if` (shift-reduce parser convention).

```
       if-statement
          /     \
         /       \
       if      else-statement
        |           |
    condition   (x = 2)
        |
       (b)
```

## 1.9 Error Handling

Robust error handling is essential for a usable compiler.

### 1.9.1 Error Categories

| Category | Phase | Examples |
|----------|-------|----------|
| **Lexical Errors** | Scanner | Invalid characters, unclosed strings |
| **Syntax Errors** | Parser | Missing semicolons, unbalanced parentheses |
| **Semantic Errors** | Analyzer | Undefined variables, type mismatches |
| **Logical Errors** | (Runtime) | Infinite loops, wrong algorithms |

### 1.9.2 Error Recovery Strategies

```cpp
// Panic mode: Skip tokens until synchronization point
void Parser::panicMode() {
    // Skip tokens until we find a synchronizing token
    while (currentToken().type != TokenType::SEMICOLON &&
           currentToken().type != TokenType::RBRACE &&
           currentToken().type != TokenType::END_OF_INPUT) {
        advance();
    }
    
    // Skip the synchronizing token
    if (currentToken().type != TokenType::END_OF_INPUT) {
        advance();
    }
}
```
