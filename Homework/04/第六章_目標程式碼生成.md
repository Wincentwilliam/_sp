## 6.1 Introduction to Code Generation

Code generation is the final phase of compilation, transforming the optimized intermediate representation into executable machine code for a specific target architecture.

### 6.1.1 Code Generator Responsibilities

```
┌────────────────────────────────────────────────────────────────────────┐
│                  CODE GENERATOR RESPONSIBILITIES                        │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │                   INSTRUCTION SELECTION                       │   │
│   │   • Map IR instructions to target machine instructions        │   │
│   │   • Choose best instruction variant                          │   │
│   │   • Handle instruction patterns and addressing modes          │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │                   REGISTER ALLOCATION                         │   │
│   │   • Assign virtual registers to physical registers            │   │
│   │   • Handle register spills when registers exhausted           │   │
│   │   • Minimize memory traffic                                  │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │                   INSTRUCTION SCHEDULING                      │   │
│   │   • Order instructions for pipeline efficiency                │   │
│   │   • Avoid pipeline stalls                                    │   │
│   │   • Maximize instruction-level parallelism                  │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │                   CALLING CONVENTIONS                        │   │
│   │   • Parameter passing (registers vs stack)                   │   │
│   │   • Return value handling                                    │   │
│   │   • Caller/callee saved registers                            │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │                   STACK FRAME MANAGEMENT                     │   │
│   │   • Local variable allocation                               │   │
│   │   • Spilled register storage                                 │   │
│   │   • Saved registers                                         │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

## 6.2 LLVM CodeGen Architecture

### 6.2.1 Target Description

```
┌────────────────────────────────────────────────────────────────────────┐
│                    TARGET DESCRIPTION SYSTEM                            │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │                    REGISTER DEFINITION                         │   │
│   │                                                              │   │
│   │   def RAX : Register<...> {                                   │   │
│   │     let namespace = "X86";                                   │   │
│   │     let encoding = ...;                                      │   │
│   │   }                                                          │   │
│   │                                                              │   │
│   │   def GR64 : RegisterClass<"X86", [i64]> {                   │   │
│   │     let members = [RAX, RBX, RCX, RDX, ...];               │   │
│   │   }                                                          │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │                    INSTRUCTION DEFINITION                     │   │
│   │                                                              │   │
│   │   def ADD64rr : X86Inst<...> {                               │   │
│   │     let Pattern = [(set GR64:$dst, (add GR64:$src1,         │   │
│   │                                   GR64:$src2))];             │   │
│   │   }                                                          │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │                    CALLING CONVENTION                         │   │
│   │                                                              │   │
│   │   def CalcArgs : CalleeSavedRegs<(sequence ...)>;           │   │
│   │   let ArgumentStack = ...;                                  │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

## 6.3 Instruction Selection

### 6.3.1 SelectionDAG

LLVM uses a Directed Acyclic Graph (DAG) representation for instruction selection:

```llvm
; LLVM IR
%result = add i32 %a, %b

; SelectionDAG representation
      ADD
      /  \
    %a   %b

; After instruction selection (x86-64)
movl  %edi, %eax    ; Load %a (first argument) to %eax
addl  %esi, %eax    ; Add %b (second argument)
```

### 6.3.2 Pattern Matching

```tablegen
// X86InstrInfo.td - Pattern matching examples

// Two-address to three-address conversion
def ADD32rr : I<0x01, MRMDestReg,
  (outs GR32:$dst), (ins GR32:$src1, GR32:$src2),
  "addl {$src2, $src1|$src1, $src2}",
  [(set GR32:$dst, (add GR32:$src1, GR32:$src2))]>;

// Immediate operand
def ADD32ri : I<0x81, 0xC0,
  (outs GR32:$dst), (ins GR32:$src1, i32imm:$src2),
  "addl ${src2}, $src1",
  [(set GR32:$dst, (add GR32:$src1, i32imm:$src2))]>;
```

## 6.4 Register Allocation

### 6.4.1 The Problem

```
┌────────────────────────────────────────────────────────────────────────┐
│                    REGISTER ALLOCATION PROBLEM                          │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│   Virtual Registers (unlimited):                                        │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │   %1 = ...        %2 = ...        %3 = ...        %4 = ...  │   │
│   │   %5 = add %1, %2  %6 = mul %3, %4   %7 = sub %5, %6      │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
│   Physical Registers (limited):                                         │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │   RAX  │  RBX  │  RCX  │  RDX  │  RSI  │  RDI  │  R8  │ R9  │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
│   Challenge: Map unlimited virtual registers to limited physical regs │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

### 6.4.2 Allocation Strategies

| Strategy | Pros | Cons | LLVM Option |
|---------|------|------|-------------|
| **Local** | Fast | Limited optimization | `fast` |
| **Greedy** | Good quality, fast | May not be optimal | `greedy` (default) |
| **Basic** | Simple | Less efficient | `basic` |
| **PBESS** | Optimal | Very slow | `pbes` |

### 6.4.3 Spilling

When physical registers are exhausted, values are "spilled" to memory:

```asm
; Without spilling
addl  %eax, %ebx        ; Fast, register-to-register

; With spilling
movl  %eax, -4(%rbp)    ; Spill %eax to stack
movl  %ebx, %eax        ; Use %eax for %ebx
addl  -4(%rbp), %eax    ; Add with spilled value
movl  %eax, -4(%rbp)    ; Spill result back
```

## 6.5 Calling Conventions

### 6.5.1 System V x86-64

```
┌────────────────────────────────────────────────────────────────────────┐
│                  SYSTEM V x86-64 CALLING CONVENTION                    │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│   PARAMETER PASSING:                                                   │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │   Arg 1  │   Arg 2  │   Arg 3  │   Arg 4  │   Arg 5  │ Arg 6 │   │
│   │   RDI    │   RSI    │   RDX    │   RCX    │   R8     │  R9   │   │
│   └──────────────────────────────────────────────────────────────┘   │
│   │                                                                 │   │
│   │   Additional arguments: Stack (right to left)                   │   │
│   │   Return value: RAX (or RDX:RAX for 128-bit)                   │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
│   REGISTER PRESERVATION:                                               │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │   Caller-Saved (Volatile)        │   Callee-Saved (Non-volatile) │   │
│   │   RAX, RCX, RDX, RSI, RDI,      │   RBX, RBP, R12, R13, R14,    │   │
│   │   R8, R9, R10, R11              │   R15                           │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
│   STACK FRAME:                                                         │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │   [Return Address]                  ← RSP                    │   │
│   │   [Saved RBP]                                             │   │
│   │   [Local Variables]                                        │   │
│   │   [Spilled Registers]                                      │   │
│   │   [Outgoing Arguments]           ← RBP                       │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

### 6.5.2 Function Prologue/Epilogue

```asm
; Function prologue
pushq  %rbp                    ; Save old frame pointer
movq   %rsp, %rbp              ; Set new frame pointer
subq   $32, %rsp              ; Allocate local space (16-byte aligned)

; Function epilogue
addq   $32, %rsp              ; Deallocate locals
popq   %rbp                   ; Restore frame pointer
ret                            ; Return to caller
```

## 6.6 Stack Frame Layout

### 6.6.1 x86-64 Stack Frame

```
┌────────────────────────────────────────────────────────────────────────┐
│                    STACK FRAME LAYOUT (x86-64)                        │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│   Higher Addresses                                                      │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │                                                              │   │
│   │   Argument N         │   Passed by caller                    │   │
│   │   ...               │                                      │   │
│   │   Argument 7         │   (if more than 6 args)              │   │
│   │   Argument 6         │   RCX / R9                           │   │
│   │   Argument 5         │   R8                                 │   │
│   │   Argument 4         │   RDX                                │   │
│   │   Argument 3         │   RSI                                │   │
│   │   Argument 2         │   RDI                                │   │
│   │   Argument 1         │   (already in RDI)                   │   │
│   ├──────────────────────────────────────────────────────────────┤   │
│   │   Return Address      │   Pushed by CALL instruction         │   │
│   ├──────────────────────────────────────────────────────────────┤   │
│   │   Saved RBP            │   pushq %rbp                        │   │
│   ├──────────────────────────────────────────────────────────────┤   │
│   │                                                              │   │
│   │   Local Variables      │   subq $N, %rsp                     │   │
│   │   Spilled Registers     │   (with alignment)                  │   │
│   │   Outgoing Args         │   (for functions we call)          │   │
│   │                                                              │   │
│   ├──────────────────────────────────────────────────────────────┤   │
│   │   [Red Zone]           │   128 bytes below RSP (SysV ABI)    │   │
│   │                       │   Not touched by signal handlers      │   │
│   └──────────────────────────────────────────────────────────────┘   │
│                                                                        │
│   Lower Addresses                                                      │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

## 6.7 Target-Specific Considerations

### 6.7.1 x86/x86-64

```asm
; x86-64 instruction examples
movq  %rdi, %rax          ; Move 64-bit register
addl  %eax, %eax          ; 32-bit operations zero-extend
mulq  %rsi                ; Multiply: RAX = RAX * RSI
cmpq  $0, %rcx            ; Compare with immediate
je    .L1                  ; Jump if equal
callq *%rax               ; Indirect call via register
```

### 6.7.2 ARM/AArch64

```asm
; AArch64 instruction examples
mov  x0, x1               ; Move register
add  x0, x0, x2           ; Add: x0 = x0 + x2
mul  x0, x1, x2           ; Multiply: x0 = x1 * x2
cbz  x0, .L1              ; Compare and branch if zero
bl   function             ; Branch with link (call)
ret                        ; Return
```

### 6.7.3 RISC-V

```asm
; RISC-V (RV64) instruction examples
add  a0, a0, a1           ; a0 = a0 + a1
mul  a0, a0, a1            ; a0 = a0 * a1
beq  t0, t1, .L1           ; Branch if equal
jal  ra, function          ; Jump and link (call)
ret                        ; Return (pseudo-instruction)
```

## 6.8 Cross-Compilation

### 6.8.1 Target Specification

```bash
# Specify target triple
clang --target=aarch64-linux-gnu -o hello_arm hello.c

# Compile for RISC-V
clang --target=riscv64-unknown-elf -o hello_riscv hello.c

# Compile for WebAssembly
clang --target=wasm32 -o hello.wasm hello.c

# Windows cross-compilation from Linux
x86_64-w64-mingw32-clang -o hello.exe hello.c
```

### 6.8.2 Cross-Compilation Workflow

```
┌────────────────────────────────────────────────────────────────────────┐
│                    CROSS-COMPILATION WORKFLOW                           │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│   BUILD MACHINE (x86-64 Linux)                                         │
│        │                                                               │
│        │  clang --target=aarch64-linux-gnu                            │
│        ▼                                                               │
│   ┌──────────────────────────────────────────────────────────────┐   │
│   │   LLVM IR                                                     │   │
│   │   ├─── Optimization passes (same on all platforms)            │   │
│   │   └─── Code generation (ARM-specific)                        │   │
│   └──────────────────────────────────────────────────────────────┘   │
│        │                                                               │
```
