### 7.2.2 LLVM Tool Reference

```bash
# Compilation
clang -S -emit-llvm -O2 input.c -o input.ll
llvm-as input.ll -o input.bc
llc -O2 input.ll -o input.s

# Optimization
opt -S -O2 input.ll -o optimized.ll
opt -S -passes=gvn,dce input.ll

# Inspection
llvm-dis < input.bc
clang -Xclang -ast-dump -fsyntax-only input.c

# Linking
llvm-link file1.bc file2.bc -o combined.bc
```

## 7.4 Advanced Topics

### 7.4.1 Topics for Further Study

- **Loop Optimization**: Unrolling, vectorization, interchange
- **Data Flow Analysis**: Reaching definitions, liveness
- **Garbage Collection**: Mark-sweep, generational collectors
- **JIT Compilation**: Runtime optimization, hot code replacement
- **Profile-Guided Optimization**: Feedback-directed compilation
- **Whole Program Optimization**: Link-time optimization (LTO)

### 7.4.2 LLVM Extensions

- **MLIR**: Multi-Level IR for heterogeneous computing
- **Clang Plugins**: Extend the C/C++ compiler
- **JIT Compilation**: LLVM's ORC JIT API
- **WebAssembly**: Compile to Wasm targets

## 7.5 Final Exercises

### Exercise 7.1: Complete Compiler ⭐⭐⭐

**Objective:** Build a full compiler

**Task:** Implement a compiler for this language:

```
// Example program
int fibonacci(int n) {
    if (n <= 1) {
        return n;
    }
    return fibonacci(n - 1) + fibonacci(n - 2);
}

int main() {
    int result = fibonacci(10);
    return result;
}
```

Requirements:
- Lexer and parser
- Semantic analysis with type checking
- LLVM IR generation
- Working executable output

### Exercise 7.2: Optimization Analysis ⭐⭐

**Objective:** Analyze optimization effects

**Task:** 
1. Write a function with optimization opportunities
2. Compile at -O0, -O1, -O2, -O3
3. Compare IR and assembly outputs
4. Document all transformations

**Objective:** Compare code generation across targets

4. Analyze differences in calling conventions
