## A.8 Glossary

### A.8.1 Compilation Terms

| Term | Definition |
|------|------------|
| **AST** | Abstract Syntax Tree - Hierarchical representation of program structure |
| **IR** | Intermediate Representation - Abstract machine code between source and target |
| **SSA** | Static Single Assignment - IR form where each variable is assigned once |
| **CFG** | Control Flow Graph - Graph of program control flow |
| **DAG** | Directed Acyclic Graph - Directed graph without cycles |
| **LTO** | Link-Time Optimization - Optimization across compilation units |

### A.8.2 LLVM Terms

| Term | Definition |
|------|------------|
| **Module** | Top-level container for all IR |
| **Function** | Callable procedure with basic blocks |
| **BasicBlock** | Sequence of instructions with single entry/exit |
| **Value** | Base class for all IR values |
| **Instruction** | Operation in a basic block |
| **Pass** | Transformation or analysis pass |

### A.8.3 Architecture Terms

| Term | Definition |
|------|------------|
| **ABI** | Application Binary Interface - Binary calling conventions |
| **Calling Convention** | How parameters are passed and values returned |
| **Register Class** | Group of registers used for similar purposes |
| **Calling Convention** | Rules for function calls |

## A.9 Quick Reference Cards

### A.9.1 LLVM IR Cheat Sheet

```llvm
; Types
i32, float, double, i8*, [10 x i32], <4 x float>

; Constants
@global = global i32 42
@str = constant [5 x i8] c"hello\00"

; Functions
define i32 @foo(i32 %a, i32 %b) {
entry:
  %sum = add i32 %a, %b
  ret i32 %sum
}

; Control flow
br label %dest
br i1 %cond, label %then, label %else
switch i32 %val, label %default [i32 0, label %zero]

; Memory
%ptr = alloca i32
%val = load i32, i32* %ptr
store i32 %val, i32* %ptr
%gep = getelementptr i32, i32* %ptr, i32 1
```

### A.9.2 Clang Compilation Flags

```bash
# Basic
clang -c file.c              # Compile to object
clang -S file.c              # Compile to assembly
clang -o prog file.c         # Compile and link

# Optimization
clang -O0 file.c             # No optimization
clang -O1 file.c             # Basic
clang -O2 file.c             # Standard
clang -O3 file.c             # Aggressive

# IR Generation
clang -S -emit-llvm file.c   # Generate .ll
clang -c -emit-llvm file.c   # Generate .bc

# Warnings
clang -Wall -Wextra file.c   # All warnings
clang -Werror file.c         # Warnings as errors

# Debug
clang -g file.c              # Debug info
clang -g -O0 file.c          # Debug + no optimize
```

### A.9.3 LLVM Tool Commands

```bash
# Assembly/Disassembly
llvm-as file.ll -o file.bc
llvm-dis file.bc -o file.ll

# Linking
llvm-link a.bc b.bc -o combined.bc

# Optimization
opt -O2 file.ll -o opt.ll
opt -S -gvn file.ll -o file_gvn.ll
opt -S -mem2reg file.ll -o file_ssa.ll

# Code generation
llc -O2 file.ll -o file.s
llc -O2 -march=arm64 file.ll -o file_arm.s

# Inspection
opt -S -dot-cfg file.ll      # CFG visualization
llvm-dis < file.bc | head     # Quick view
```
