## 8.6 Glossary

### 8.6.1 Compilation Terms

| Term | Definition |
|------|------------|
| **AST** | Abstract Syntax Tree - Hierarchical program structure |
| **IR** | Intermediate Representation - Abstract machine code |
| **SSA** | Static Single Assignment - Each variable assigned once |
| **CFG** | Control Flow Graph - Program control flow representation |
| **DAG** | Directed Acyclic Graph - Directed graph without cycles |
| **LTO** | Link-Time Optimization - Optimization across translation units |

### 8.6.2 LLVM Terms

| Term | Definition |
|------|------------|
| **Module** | Top-level container for all IR |
| **Function** | Callable procedure |
| **BasicBlock** | Single-entry/exit instruction sequence |
| **Value** | Base class for IR values |
| **Pass** | Transformation or analysis |

## 8.7 Quick Reference

### 8.7.1 LLVM IR Cheat Sheet

```llvm
; Types
i32, float, double, i8*, [10 x i32], <4 x float>

; Constants
@global = global i32 42
@str = constant [5 x i8] c"hello\00"

; Functions
define i32 @foo(i32 %a, i32 %b) {
entry:
  %sum = add i32 %a, %b
  ret i32 %sum
}

; Control flow
br label %dest
br i1 %cond, label %then, label %else
ret i32 %val
```

### 8.7.2 Clang Compilation Flags

```bash
# Basic
clang -c file.c              # Compile to object
clang -S -emit-llvm file.c   # Generate .ll
clang -o program file.c       # Compile and link

# Optimization
clang -O0 file.c             # No optimization
clang -O2 file.c             # Standard optimization
clang -O3 file.c             # Aggressive optimization

# IR Generation
clang -S -emit-llvm -O2 file.c -o file.ll
```
